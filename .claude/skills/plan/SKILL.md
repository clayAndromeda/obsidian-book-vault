# 実装計画スキル

引数に応じて3つのフェーズを実行する。

## 使い方

- `/plan <機能名>` — 新しい計画を作成
- `/plan done` — 現在の計画を完了として片付ける

---

## `/plan <機能名>` — 計画作成フェーズ

1. プランモードに入る（EnterPlanMode）
2. ユーザーと対話しながら実装計画を練る
3. 合意が取れたら `docs/plan-<機能名>.md` に計画を書き出す
   - 内容: 概要、実装ステップ（チェックボックス付き）、影響するファイル、検証方法
4. CLAUDE.md の末尾に以下を追加:
   ```markdown
   ## 進行中の作業
   現在 [<機能名>の計画](docs/plan-<機能名>.md) を実装中。
   ```
5. プランモードを抜ける（ExitPlanMode）
6. ユーザーに計画をレビューしてもらい、OKなら実装開始

### 計画ファイルのテンプレート

```markdown
# <機能名>

## 概要
<この機能が何をするか、なぜ必要か>

## 実装ステップ
- [ ] ステップ1
- [ ] ステップ2
- [ ] ...

## 影響するファイル
- `src/xxx.ts` — 変更内容

## 検証方法
- どうやって動作確認するか
```

---

## `/plan done` — 完了フェーズ

1. `docs/plan-*.md` を確認し、現在の計画ファイルを特定
2. 計画ファイルを削除
3. CLAUDE.md から「## 進行中の作業」セクションを削除
4. メモリに完了記録を保存（何を実装したか、学んだことがあれば）
5. ユーザーに完了を報告
